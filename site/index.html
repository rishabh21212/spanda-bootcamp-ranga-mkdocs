    Detailed Contents

    Pre-Labs 1-3: Docker, Git, Github, Python, Intro to Data Science, ML and DL

    We review some prerequisites -- the DNN architectures we'll be using and basic model training with PyTorch -- and introduce PyTorch Lightning. Published March 10, 2024.

    Lecture 1: Course Vision and Running a Local LLM

    We review the purpose of the course and consider when it's a good (or bad!) idea to use ML. Published March 8, 2024.

    Lab Overview

    We walk through the entire architecture of the application we will be building, from soup to nuts. Published February 25, 2024.

    Lecture 2: Software Engineering for AI

    We tour the landscape of infrastructure and tooling for developing deep learning models. Published March 15, 2024.

    Lab 4: Experiment Management

    We run, track, and manage model development experiments with Weights & Biases. Published March 17, 2024.

    Lecture 3: Ops - DevOps, MLOps, AIOps

    We look at tools and practices for testing software in general and ML models in particular. Published March 22, 2024.

    Lab 5: Troubleshooting & Testing

    We try out some Python testing tools and dissect a PyTorch trace to learn performance troubleshooting techniques. Published March 24, 2024.

    Lecture 4: Data Collection, Data Labeling, Data Management, Analytics

    We look at sourcing, storing, exploring, processing, labeling, and versioning data for deep learning. Published March 29, 2024.

    Lab 6: Data Annotation

    We spin up a data annotation server and learn just how messy data really is. Published March 31, 2024.

    Lecture 5: Testing and Deployment

    We do a lightning tour of all the ways models are deployed and do a deep dive on running models as web services. Published April 5, 2024.

    Lab 7: Web Deployment

    We create and deploy our ML-powered text recognition application with a simple web UI and a serverless model service. Published April 7, 2024.

    Lecture 6: LLM Serving and Reinforcement Learning

    We consider what it takes to build a continual learning system around an ML-powered application. Published April 12, 2024.

    Lab 8: Model Monitoring

    We add user feedback to our ML application and review data logged by actual users of the FSDL Text Recognizer. Published April 14, 2024.

    Lecture 7: Distributed Systems Patterns

    We look at how to build on GPT-3, CLIP, StableDiffusion, and other large models. Published April 19, 2024.

    Lecture 8: Workflow Patterns

    We look at the structure of ML teams and projects, including how to hire or get hired on an ML team and how to build an ML-first organization. Published April 26, 2024.

    Lecture 9: Scheduling & Metadata Patterns

    We consider ethical concerns around buiding technlogy, building with machine learning, and building artificial intelligence. Published May 3, 2024.

    Course Creators

    This course was only possible with the support of our amazing Staff (in no particular order):

    • Andrew Mendez is a Deep Learning Solutions Engineer at DeterminedAI, working on computer vision and NLP solutions for defense and autonomous vehicle companies. Previously Andrew worked as an ML Engineer at Clarifai and CACI.
    • Daniel Hen is a Senior Data Scientist at Digital Turbine, working on Ad Tech and mobile solutions, as well as Big Data problems. Working with Spark, ML algorithms such as XGBoost, Computer Vision, and constantly learning new technology.
    • James Le runs Data Relations and Partnerships at Superb AI, a data management platform for computer vision use cases. Outside work, he writes data-centric blog posts, hosts a data-focused podcast, and organizes in-person events for the data community.
    • Saurabh Bipin Chandra is a Senior ML Scientist at Turnitin.
    • Sayak Paul is as a Machine Learning Engineer at Carted on NLP and representation learning from HTML webpages. Besides work, he contributes to various open-source projects.
    • Vishnu Rachakonda is a Data Scientist at firsthand.
